## 为什么要梳理

博客上线一段时间了，功能陆续在加——文章展示、摄影墙、股票观察、便利贴。代码也写得越来越乱。趁还没变成屎山，先把整体结构理一遍，不然过段时间又得捋。

后面打算优化一下性能，提升一下后端安全性，万一哪天被脚本扫到就不好了。

## 整体结构

```
浏览器
   ↓
Nginx (端口 80/443)
   ↓  
├─ /             → Vue 3 静态文件
├─ /api/*        → Spring Boot (端口 8080)
└─ /ws           → Spring Boot WebSocket
   ↓
├─ MySQL 8.0     → 文章、评论、便利贴
├─ Redis          → Session、缓存
└─ 新浪 API       → 股票行情
```

跑在一台轻量服务器上，Docker Compose 管理四个容器。内存 4G，属于是绰绰有余了。

## 前端：Vue 3 + Vite + Tailwind

前端这块选了 Vue 3，用了 Composition API。组件不算多，结构大概是这样：

```
src/
├── views/         # 页面级组件
│   ├── HomeView.vue       # 首页
│   ├── EssaysView.vue     # 随笔列表
│   ├── ArticleView.vue    # 文章详情
│   ├── PhotographyView.vue # 摄影
│   ├── FinanceView.vue    # 股票观察
│   └── PhotoDetailView.vue
├── components/    # 可复用组件
│   ├── MarkdownRenderer.vue
│   ├── PhotoLightbox.vue
│   └── EssayCard.vue
├── composables/   # 组合式函数
│   ├── usePosts.js
│   └── usePhotos.js
├── stores/        # Pinia 状态管理
├── router/        # Vue Router
└── api.js         # Axios 封装
```

Markdown 渲染用的是 `markdown-it`，数学公式用的 `KaTeX`，代码高亮用的 `highlight.js`。图表用的 ECharts（不过目前还没加图表的页面，先装上占个坑）。

Tailwind CSS 4 默认的 utility class 写法还是挺爽的，比如写一个卡片：

```html
<div class="bg-white rounded-2xl border border-black/5 p-5">
```

不用自己写一堆 CSS。但有些复杂的定制样式还是得回到 scoped style，比如那个鼠标跟随的光晕效果。

## 后端：Spring Boot 3

后端就简单多了，基本上就是个数据转发层。

Controller 就几个：

```
/api/posts          → 文章 CRUD
/api/photos         → 摄影作品
/api/finance/*      → 股票数据（转发新浪API）
/api/market-notes   → 便利贴 CRUD
/api/auth           → 登录（后面再加）
```

没有用 MyBatis 也没有用 JPA，直接 JDBC Template 手写 SQL。因为表本身很少，写的 SQL 也不复杂。

股票数据那块最折腾，费老劲新浪的免费 API 有各种细节问题——不同前缀的字段格式不一样、休市时数据清零、有些指数压根没有，后面应该会替换。上次专门写了篇踩坑记录，这里就不重复了。

## 数据库设计

暂时有点简陋：

```
posts           # 文章
  id, title, content, summary, tags, created_at, updated_at

photos          # 摄影作品
  id, title, url, thumbnail, description, taken_at

market_notes    # 便利贴（股票快评）
  id, title, content, asset_code, color, created_at, updated_at

users           # 用户（目前就我自己）
  id, username, password_hash
```

没有外键约束，因为目前就一个用户，也不需要复杂的关联查询。

## 部署：Docker Compose

四个容器，一个 `docker-compose.yml` 管所有：

```yaml
services:
  db:       # MySQL 8.0
  redis:    # Redis
  backend:  # Spring Boot
  frontend: # Nginx + Vue 静态文件
```

MySQL 和 Redis 的数据挂载到宿主机目录，做了持久化。后端容器用的是多阶段构建——先在 Maven 镜像里编译打包，再把 jar 拷到一个干净的 JRE 镜像里跑。前端也是类似的，Node 镜像里 build，然后把 dist 丢进 Nginx。

CI/CD 没有，因为我每次改完都是手动 `docker build` + `docker-compose up -d`。够用了。
