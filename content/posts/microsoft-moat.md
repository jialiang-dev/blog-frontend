## 写在前面

最近研究微软这家公司。

Honestly，一开始我对微软的印象还停留在 Windows 和 Office，买它的原因是因为它对世界的改变和在我日常生活中的渗透率，第一印象就是一家卖系统和办公软件的公司，外带Azure业务。但越看财报和资料，越觉得这家公司比我想象中复杂得多。

从个人PC到云计算时代，再到现在的 AI 热潮，微软总能找到自己的位置。所以我想试着记录一下，目前我理解的微软护城河到底是什么。

当然，我还只是个学习中的投资者，很多地方可能理解得不够深入，这篇文章更像是学习笔记。

## 第一层护城河：企业离不开它

很多大型企业使用的服务，可能大半都来自微软。

直观一些的，比如员工每天要用：

* Windows
* Outlook
* Word
* Excel
* Teams

而不直观的，公司后台可能在用：

* Azure 云服务
* GitHub 企业版
* Dynamics 365

这些产品市面上竞品众多，也并非所有企业都在使用。但事实是，自Windows系统发布以来，它几乎成为全世界电脑运行的标准配置。即便Mac OS以简洁高效赢得了一部分市场份额，几十年的使用惯性，加上深入人心的品牌积累，依然让绝大多数人牢牢绑定在Windows生态之中。

就算一家颇有规模的公司打算进行系统转换，用了十几年微软的系统，员工培训、数据迁移、业务流程重建都会产生巨大的成本。这基本上杜绝了竞品对老牌公司的渗透。

理论上能换，实际上很难换。

这也是为什么微软的收入一直比较稳定，因为很多企业客户已经深度绑定在它的生态里面了。

## 第二层护城河：开发者生态

作为计算机专业的人，我对这一点感受比较深。

* 开源 .NET
* 收购 GitHub
* 推出 VS Code
* 推广 TypeScript

特别是 VS Code和GitHub，这是开发者的标配。

虽然 VS Code 本身不赚钱，但它能让越来越多开发者习惯微软的工具和生态。

先让开发者愿意用你的工具，以后企业做技术选型的时候，自然更容易选择你的平台。

## 第三层护城河：AI

最近最热门的话题肯定是 AI。

很多公司都在做 AI，但微软有一个比较特殊的优势，就是它不仅参与了 AI，还把 AI 塞进了自己的各种产品里。

比如：

| 产品      | AI 功能        |
| ------- | ------------ |
| GitHub  | Copilot 编程助手 |
| Office  | 文档和表格助手      |
| Azure   | 提供 AI 服务     |
| Windows | 系统级 Copilot  |
| Bing    | AI 搜索        |

我觉得，微软最大的优势不是拥有某一个 AI 产品，而是拥有大量已经在付费的用户。

当企业本来就在用 Office、Azure、Windows 的时候，微软推广 AI 服务会比很多竞争对手轻松得多。

> 但不能忽视的一点是，Copilot即使依靠Windows的强大生态，其在AI热潮中的存在感依旧较低。我认为其当下最大的风险点在于，微软想做AI时代的卖铲人——通过Azure云服务和巨额的数据中心投入，试图在别人都想“卖电”时，“包下煤矿和河流”。这是战略也是豪赌：如果未来AI的应用层爆发不足，或算力，硬件性能有巨大技术突破，那微软巨资打造的算力基础设施就可能面临需求不足或议价能力被削弱的风险，盖的数据中心变成疫情爆发前的新盖楼盘。

## 最后

搜完资料后，我越来越觉得微软的护城河并不是某一个产品。

Windows 很强，但微软不只是 Windows。

Office 很强，但微软也不只是 Office。

它在过去，现在，和将来都有巨大的成就和潜力。

如果未来几年 AI 真的会改变很多行业，那么微软大概率会是受益者之一。

当然，这不代表微软股价一定会涨，也不代表它没有风险。

但至少从商业模式，基本面来看，它是一家值得长期跟踪研究的优质资产。
